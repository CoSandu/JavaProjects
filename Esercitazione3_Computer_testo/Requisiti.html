Computer


Progettare ed implementare un programma che permetta di gestire cartelle e file di un computer. Tutte le classi devono appartenere al package computer.

Il programma interagisce attraverso istanze della classe Computer.

R1. Computer e cartelle

La classe principale con cui interagire è Computer, che riceve come parametro del costruttore la dimensione dell'hard disk in KB (si assuma che un computer possa avere un solo hard disk). È possibile accedere a tale informazione tramite il metodo getter getDimensioneHardDisk().
Il computer può memorizzare cartelle e file. Alla creazione, ogni computer deve essere automaticamente dotato di una cartella con nome root, che conterrà al suo interno tutte le cartelle e sotto-cartelle successivamente create. Tale cartella è accessibile tramite il metodo getRoot().
Per creare una cartella, si utilizza il metodo creaCartella() della classe Computer, che riceve come parametri il nome della cartella che si vuole creare e un oggetto di tipo Cartella, che rappresenta la cartella all'interno della quale verrà salvata la cartella appena creata. Tale metodo deve permettere la creazione di una cartella se e solo se all'interno della cartella contenitore non esiste nessuna cartella con nome uguale a quello passato come parametro al metodo. Qualora sia già presente una cartella con lo stesso nome, il sistema deve ignorare la richiesta.
È possibile cercare una cartella contenuta all'interno di un'altra cartella tramite il metodo cercaCartella(), che riceve come parametri il nome della cartella cercata e la cartella all'interno della quale effettuare la ricerca. Tale metodo confronta il nome della cartella cercata con quelli delle cartelle contenute nella cartella contenitore e restituisce le informazioni relative all'oggetto trovato (riportando il nome della cartella e il nome della cartella superiore separati da "-", ad esempio Documents-root) se e solo se il nome coincide. Qualora non sia identificata nessuna cartella con quel nome, il metodo restituisce una stringa vuota.
NB: all'inizio e alla fine della stampa restituita dal metodo cercaCartella non devono essere presenti spazi ne' caratteri speciali

R2. File semplici

Le cartelle possono contenere file. I file possono essere semplici, oppure speciali (cioè file che presentano informazioni aggiuntive).
I file semplici sono aggiunti a una cartella tramite il metodo creaFile() della classe Computer, che riceve come parametri il nome del file da creare, la cartella all'interno della quale verrà salvato il file creato e la dimensione in KB e restituisce l'oggetto appena creato. In maniera analoga al metodo creaCartella(), tale metodo permette di creare un file solo se non esiste nessun altro file con il nome specificato, all'interno della cartella.
Come per il metodo cercaCartella(), il metodo cercaFile() permette di cercare un file contenuto in una cartella e ottenerne le informazioni. Tale metodo restituisce una stringa contenente nome del file, nome della cartella superiore e dimensione, separati da un "-" (ad esempio documento1-Documents-10020).

R3. File speciali

Oltre ai file semplici, esistono dei file speciali (file audio e immagini) per i quali sono memorizzate informazioni aggiuntive.
In particolare, i file audio presentano informazioni sui minuti e i secondi di durata del brano (oltre alle informazioni specificate per i file semplici), mentre i file immagine memorizzano informazioni relative all'altezza e larghezza in pixel dell'immagine, e al dispositivo utilizzato per acquisire l'immagine.
Entrambi i tipi di file speciali possono essere creati e aggiunti a una cartella tramite i metodi creaFileAudio() e creaFileImmagine() rispettivamente, che ricevono come parametri, nel caso di un file audio nome del file, cartella superiore, dimensione, minuti e secondi di durata, e nel caso di una immagine, nome del file, cartella superiore, dimensione, altezza, larghezza e modello del dispositivo utilizzato.
In maniera analoga alla creazione di un file semplice, tali metodi permettono di creare un nuovo file solo se non sono presenti all'interno della cartella altri file con lo stesso nome.
Infine, come per i file semplici, il metodo cercaFile() permette di cercare un file audio o immagine contenuto in una cartella e ottenerne le informazioni. Tale metodo restituisce per i file audio una stringa contenente nome del file, nome della cartella superiore, dimensione, minuti e secondi di durata separati da un "-" (ad esempio audio1-Audio-20809-2-53), e per le immagini una stringa contenente nome del file, nome della cartella superiore, dimensione, altezza, larghezza e modello del dispositivo utilizzato separati da un "-" (ad esempio gruppo-Estate2015-8957-2304-3456-Canon EOS).

R4. Elenco file e calcolo dimensione occupata

Le informazioni relative ai file contenuti in una cartella possono essere visualizzate tramite il metodo elencoFileCartella(). Tale metodo riceve come parametro l'oggetto relativo alla cartella della quale si vogliono visualizzare i file, e restituisce una stringa contenente le informazioni relative ai file contenuti, riportati uno per riga (tutte le righe, compresa l'ultima, terminano con "\n"), nello stesso formato specificato per il metodo cercaFile().
I file vanno riportati nell'ordine di inserimento, ad esempio:
audio1-Audio-20809-2-53
audio2-Audio-32398-4-12
lista canzoni-Audio-5009

È possibile conoscere la dimensione occupata da una cartella e dal suo contenuto (file e cartelle contenuti in essa e/o nelle sue sotto-cartelle) tramite il metodo calcolaDimensioneOccupata(). Tale metodo riceve come parametro un oggetto di tipo Cartella e restituisce la dimensione occupata da essa e dal suo contenuto. Si ipotizzi che ogni cartella vuota occupi uno spazio di 10 KB.

NB: si assuma che ogni cartella non possa contenere, al suo interno più di 500 cartelle e di 500 file.
